PHASE 1: Project Foundation
Ticket ID: INIT-001
Feature Name: Next.js & Tailwind Configuration Objective: Establish the foundational frontend framework, dark-mode styling, and typography system. Description: Configure Next.js 16 App Router. Setup Tailwind CSS with oklch color variables for the GetAWay brand (Teal primary). Add Inter and Poppins fonts. Ensure the color scheme is dark and enforced globally. Acceptance Criteria:
✓ Next.js app runs without errors.
✓ Tailwind compiles custom variables from globals.css.
✓ Inter and Poppins fonts load correctly.
✓ Dark mode is the default and only theme. Dependencies: None Priority: Must Have Complexity: S Estimated Effort: 2 hours Technical Areas: Frontend, CSS Files Likely Impacted: package.json, src/app/layout.tsx, src/app/globals.css, tailwind.config.ts (or Tailwind v4 config) AI Agent Prompt:
"Initialize a Next.js 16 App Router project with Tailwind CSS. Modify globals.css to include the specific oklch dark-mode brand colors for GetAWay (Primary: oklch(0.7 0.15 160)). Load Inter and Poppins fonts in layout.tsx and apply them to standard text and headings respectively."
Ticket ID: INIT-002
Feature Name: Component Library Setup Objective: Install base UI primitives to accelerate feature development. Description: Install @base-ui/react, lucide-react, and configure base primitive components (Button, Card, Input, Badge) using cva and tailwind-merge. Acceptance Criteria:
✓ Button component supports default, outline, ghost, and destructive variants.
✓ Card component supports the .glass-panel UI aesthetic.
✓ cn() utility is functional. Dependencies: INIT-001 Priority: Must Have Complexity: M Estimated Effort: 3 hours Technical Areas: Frontend Files Likely Impacted: src/lib/utils.ts, src/components/ui/button.tsx, src/components/ui/card.tsx AI Agent Prompt:
"Create a cn utility using clsx and tailwind-merge in src/lib/utils.ts. Implement reusable React components for Button, Card, Badge, and Input using cva for variants, matching the GetAWay glassmorphism and teal aesthetic."
Ticket ID: INIT-003
Feature Name: Firebase Client SDK Setup Objective: Initialize Firebase for frontend auth and real-time database access. Description: Add firebase package. Create a lazy singleton client in src/lib/firebase/client.ts. Bind to NEXT_PUBLIC_FIREBASE_* environment variables. Acceptance Criteria:
✓ Firebase app initializes without errors on the client.
✓ Missing env vars throw clear runtime errors. Dependencies: None Priority: Must Have Complexity: S Estimated Effort: 1 hour Technical Areas: Frontend, Firebase Files Likely Impacted: src/lib/firebase/client.ts, .env.example AI Agent Prompt:
"Create a Firebase client initialization script in src/lib/firebase/client.ts. Export auth and db (Firestore). Ensure it uses the singleton pattern to prevent re-initialization during hot reloading. Bind to standard Next.js public environment variables."

PHASE 2: Database Layer
Ticket ID: DB-001
Feature Name: Firestore Collections & Zod Schemas Objective: Define strongly-typed data structures for Rooms, Sessions, and Signaling. Description: Create src/lib/schemas.ts defining Zod schemas for the Room, Session, Candidate, and TrustLog objects. Create helper functions to access these collections in Firestore. Acceptance Criteria:
✓ Zod schemas exist for all core data models.
✓ TypeScript interfaces are inferred from Zod. Dependencies: INIT-003 Priority: Must Have Complexity: M Estimated Effort: 3 hours Technical Areas: Firestore, Types Files Likely Impacted: src/lib/schemas.ts, src/lib/db.ts AI Agent Prompt:
"Create Zod schemas and inferred TypeScript types in src/lib/schemas.ts for a Room (roomId, createdAt, recruiterId), Session (id, roomId, status, alerts), CandidateInfo (name, email), and WebRTC Signal data. Ensure strict typing."
Ticket ID: DB-002
Feature Name: Basic Security Rules Objective: Prevent unauthorized database reads/writes. Description: Write initial firestore.rules. Block all access by default. Allow recruiters to read/write their own rooms. Allow candidates to read room details and write signaling data using roomId. Acceptance Criteria:
✓ Recruiters cannot read other recruiters' data.
✓ Candidates can only write to the /webrtc/{roomId} subcollection. Dependencies: DB-001 Priority: Must Have Complexity: M Estimated Effort: 2 hours Technical Areas: Firestore, Security Files Likely Impacted: firestore.rules AI Agent Prompt:
"Write Firebase firestore.rules. Deny all by default. Allow recruiters to CRUD rooms where resource.data.recruiterId == request.auth.uid. Allow unauthenticated users to read rooms if they know the ID, and write to webrtc/{roomId}/* subcollections for signaling."

"Create a login page at src/app/auth/login/page.tsx using the Card, Input, and Button components. Include a form with Email and Password fields, state for loading, and inline error display. Use GetAWay's dark glassmorphism style."

✓ Successful login redirects to dashboard.

"Integrate Firebase client authentication into the login page. On form submit, call signInWithEmailAndPassword. If successful, use useRouter().push('/recruiter/overview'). Catch and display Firebase auth errors appropriately."

Description: Implement src/middleware.ts to check for a valid Firebase Auth session cookie before allowing access to /recruiter/*. Redirect to /auth/login if missing. Acceptance Criteria:
✓ Unauthenticated users visiting /recruiter/overview are redirected.
✓ Authorized users can load the dashboard. Dependencies: AUTH-002 Priority: Must Have Complexity: M Estimated Effort: 4 hours Technical Areas: Backend, Security Files Likely Impacted: src/middleware.ts, src/app/api/auth/session/route.ts AI Agent Prompt:
"Implement Next.js middleware to protect all routes under /recruiter/*. Since Firebase Client Auth uses IndexedDB, you will need to create an API route that accepts an ID token and sets a secure session cookie. Have the middleware verify this cookie."

✓ ICE candidates are exchanged and connection state reaches connected.
✓ Remote stream is successfully received. Dependencies: RTC-001, DB-002 Priority: Must Have Complexity: XL Estimated Effort: 8 hours Technical Areas: WebRTC, Firestore Files Likely Impacted: src/hooks/useWebRTC.ts AI Agent Prompt:
