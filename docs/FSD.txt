1. Executive Summary
Product Purpose
GetAWay is a browser-native, AI-powered interview proctoring platform that enables recruiters to conduct, monitor, and analyse live video interviews in real time. It uses WebRTC for P2P video, Firebase for real-time data, and browser-side AI to track trust signals — all without requiring any native installation.
Target Users
User
Role
Primary Surface
Recruiter
Creates rooms, monitors live sessions, reviews reports
/recruiter/* dashboard
Candidate
Accepts invitation, completes onboarding, participates in interview
/interview/[roomId]
Interviewer
Joins the same room URL with ?view=interviewer to see the live panel
/interview/[roomId]?view=interviewer

Design Goals
Zero friction for candidates — join an interview with a link, no account needed.
Power-dense recruiter dashboard — surface everything a recruiter needs without navigating away.
Professional, trust-conveying aesthetics — a dark, glassmorphic UI that communicates credibility.
Real-time fidelity — the UI must reflect live data (trust scores, alerts, connection state) instantly.
No cognitive overload — every screen has a single primary action.
User Experience Philosophy
Browser-first: No downloads, no plugins, no native apps.
Low friction for candidates: Room ID is the only credential. Three steps from link to live video.
Recruiter-focused dashboard: Left-rail navigation, consistent content area, glassmorphic cards.
Trust-oriented visual language: Green teal primary conveys integrity. Destructive red is reserved exclusively for warnings and errors.
Professional minimalism: Dark background eliminates noise. Content speaks for itself.

2. Design Principles
1. Clarity Over Decoration
Every UI element must earn its place. No decorative components that do not convey information. The glassmorphism aesthetic serves clarity — it creates visual layers without adding noise.
Applied in code: glass-panel utility applies backdrop-blur-md and a subtle border-white/10 — giving depth without heavy decoration.

2. Single Primary Action Per Screen
Each page has one dominant action that does not compete with other interactive elements.

3. Consistent Visual Hierarchy
Three levels of typographic weight exist on every content surface:
Primary text (text-foreground) — headings and values
Secondary text (text-muted-foreground) — labels, descriptions, captions
Accent text (text-primary) — teal for live values, KPIs, active states

4. Minimal Cognitive Load
No nested dropdowns
No multi-step modals
Confirmations use Sonner toasts, not blocking dialogs
Error states are inline, not separate screens (except terminal failures like WebRTC connection failed)

5. Progressive Disclosure
The candidate journey uses multi-step component flow (CandidateFlow) that reveals the next step only after the current one is complete. Recruiters see the room creation result only after clicking "Generate."

6. Fail States Are First-Class
Every async operation has explicit loading, success, and failure states. Connection failures, permission denials, and empty data states all have distinct UI treatments — they are never silently ignored.

7. Data Is Never Stale
All live session data (connection state, trust score, alerts) is updated in real time via Firestore listeners or local state updates. The UI should never show outdated values.

Trust Score Color Coding
This is a semantic color system used across InterviewerView and the Report page:
score > 80  → text-primary    (#10B981) // High trust — green
score > 60  → text-yellow-500 (#EAB308) // Medium trust — yellow
score ≤ 60  → text-destructive (#EF4444) // Low trust — red

Applied to: Trust score badge in InterviewerView, trust score bar in SessionsPage and ReportPage.

Color Accessibility Notes
[!WARNING] Several color combinations have not been formally audited for WCAG 2.1 compliance. The following should be verified:
text-muted-foreground (#94A3B8) on bg-card (#243044) — estimated contrast ~4.2:1 (borderline AA)
Teal primary #10B981 on dark card — estimated contrast ~3.5:1 (fails AA for normal text, passes for large text)
All body text #F8FAFC on #1E2A3A — estimated contrast ~14:1 ✅

Always pair icons with text for primary actions — never icon-only unless in a round icon button with an ARIA label.

Animate only Loader2 with animate-spin. No other icons should animate.

The red recording dot (bg-destructive animate-pulse)
